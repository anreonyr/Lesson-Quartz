# AI代码管理策略

## 习惯养成

1. **每次让AI大改之前**：先提交当前版本
2. **AI完成后**：用Lazygit查看差异
3. **定期同步**：每天开始时 `pull`，完成后 `push`
4. **实验用分支**：让AI在分支上尝试新想法

## 使用Lazygit

Lazygit是一个Git命令行工具的TUI，让操作更直观。

### 常用快捷键

| 按键 | 功能 |
|------|------|
| `s` | 暂存文件（git add） |
| `c` | 提交（git commit） |
| `p` | 推送（git push） |
| `P` | 拉取（git pull） |
| `d` | 查看差异 |
| `r` | 查看提交历史 |
| `q` | 退出 |

## 最佳实践

1. **越早越好**：项目开始就创建Git仓库
2. **频繁提交**：小改动就提交，不要等到最后
3. **写好信息**：提交信息要清楚描述改了什么
4. **注意备份**：重要改动前先确认代码已经提交
5. **学会撤销**：掌握回退的方法，有备无患
