# 拿来主义

## 为什么重要

- 不要重复造轮子
- 成熟的库经过了大量测试，比自己写的更可靠
- 使用现成方案可以大大加快开发速度

## 如何做到

- 了解有哪些现成的库/框架可以用
- 先搜索，再决定是否自己写
- 选择主流、社区活跃的库

## 示例

❌ "自己写一个HTTP服务器"
✅ "使用Flask或FastAPI来构建Web服务"

## 常见选择

| 需求 | 推荐方案 |
|------|----------|
| Web开发 | Django、Flask、FastAPI |
| 数据库 | MySQL、PostgreSQL、MongoDB |
| 缓存 | Redis |
| 异步任务 | Celery |

## 原则

- Web开发选成熟的Web框架
- 数据库选成熟的数据库
- 缓存选成熟的方案
