# 迭代器模式

**意图**：提供一种方法顺序访问集合中的元素，而不暴露其底层表示。

## 解决的问题

- **接口简化**：调用方只需用 `for ... in` 遍历，不需要知道底层结构
- **解耦**：遍历逻辑和数据结构分离，数据结构变化不影响遍历代码

## 适用场景

- 遍历复杂数据结构
- 自定义遍历逻辑

## 生活例子

就像遍历一棵树（从根到叶子），或者遍历文件系统。

## 代码示例

```python
class TreeNode:
    def __init__(self, value, children=None):
        self.value = value
        self.children = children or []

class TreeIterator:
    """树的迭代器：深度优先遍历"""
    def __init__(self, root):
        self.root = root
        self.stack = [root]

    def __iter__(self):
        return self

    def __next__(self):
        if not self.stack:
            raise StopIteration
        node = self.stack.pop()
        # 先加入右孩子，再加入左孩子（保证左孩子先被处理）
        for child in reversed(node.children):
            self.stack.append(child)
        return node.value

# 构建一颗简单的树
root = TreeNode('root', [
    TreeNode('a', [TreeNode('d'), TreeNode('e')]),
    TreeNode('b'),
    TreeNode('c')
])

# 使用 - 调用方不需要知道是树还是其他结构
print("树的深度优先遍历:")
for node_value in TreeIterator(root):
    print(node_value)
# 输出: root, a, d, e, b, c
```
