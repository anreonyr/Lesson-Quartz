# 单体架构

所有功能打包在一个应用中。

## 优点

- 简单，适合小项目
- 部署简单
- 开发、调试方便

## 缺点

- 代码多了难以维护
- 难以扩展
- 技术栈难以更换

## 适用场景

- 个人项目、小工具
- 学习用途
- 初期快速原型

## 项目结构示例

```
mytool/
├── main.py        # 入口
├── utils.py       # 工具函数
└── data.json      # 数据文件
```
