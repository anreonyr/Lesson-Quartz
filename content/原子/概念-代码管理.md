# 代码管理

代码管理是指对代码版本进行跟踪和管理的系统。

## 常见工具

| 工具 | 类型 | 特点 |
|------|------|------|
| Git | 分布式 | 最流行，功能强大 |
| SVN | 集中式 | 简单，适合旧项目 |
| Mercurial | 分布式 | 简单易用 |

## Git 核心操作

| 操作 | 命令 |
|------|------|
| 初始化 | `git init` |
| 克隆 | `git clone <url>` |
| 添加文件 | `git add <file>` |
| 提交 | `git commit -m "message"` |
| 推送 | `git push` |
| 拉取 | `git pull` |
| 创建分支 | `git branch <name>` |
| 切换分支 | `git checkout <name>` |

## 为什么需要代码管理

- **版本追溯**：随时回到任意历史版本
- **协作开发**：多人并行开发不冲突
- **分支管理**：新功能在独立分支开发
- **备份同步**：代码安全存储在远程仓库
