# 工厂模式

**意图**：定义创建对象的接口，让子类决定实例化哪个类。

## 解决的问题

- **接口简化**：调用方只需调用 `get_parser('pdf')`，不需要知道具体创建哪个类
- **解耦**：新增文件类型时，调用方代码不需要修改

## 适用场景

- 创建不同类型的文件处理器
- 创建不同的支付方式
- 根据配置创建不同的数据源

## 生活例子

就像手机数据线的充电器工厂，生产苹果口、安卓口、Type-C口的数据线。

## 代码示例

```python
class PDFParser:
    def parse(self, content):
        return "解析PDF文件"

class WordParser:
    def parse(self, content):
        return "解析Word文件"

class ExcelParser:
    def parse(self, content):
        return "解析Excel文件"

class ParserFactory:
    @staticmethod
    def get_parser(file_type):
        parsers = {
            'pdf': PDFParser,
            'docx': WordParser,
            'xlsx': ExcelParser
        }
        parser_class = parsers.get(file_type.lower())
        if parser_class:
            return parser_class()
        raise ValueError(f"不支持的文件类型: {file_type}")

# 使用 - 调用方只需知道类型字符串，不需要知道具体类
parser = ParserFactory.get_parser('pdf')
print(parser.parse("内容"))  # 输出: 解析PDF文件
```
