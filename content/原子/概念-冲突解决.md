# 冲突解决

当多人协作或合并分支时，可能会出现冲突。

## 冲突原因

- 同一行代码被多人修改
- 一个文件被删除，另一个对它进行了修改

## 解决步骤

### 1. 查看冲突文件

```bash
git status
```

### 2. 手动解决冲突

打开冲突文件，会看到类似：

```<<<<<<< HEAD
当前分支的代码
=======
要合并的代码
>>>>>>> 分支名
```

手动选择保留哪部分，或者合并两者。

### 3. 标记解决

```bash
git add 冲突文件
git commit -m "解决冲突"
```

## 避免冲突

- 经常 `pull` 最新代码
- 合并前先 `fetch` 并处理
- 及时提交自己的修改
