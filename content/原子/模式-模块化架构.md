# 模块化架构

按功能模块划分，但仍在同一个项目中。

## 项目结构

```
myproject/
├── users/          # 用户模块
│   ├── models.py
│   ├── views.py
│   └── __init__.py
├── orders/         # 订单模块
│   ├── models.py
│   ├── views.py
│   └── __init__.py
└── shared/         # 共享代码
    ├── utils.py
    └── constants.py
```

## 优点

- 模块之间低耦合
- 便于团队协作
- 可以复用模块

## 缺点

- 模块边界需要设计好
- 依赖管理要小心

## 适用场景

- 中型项目
- 多人协作
