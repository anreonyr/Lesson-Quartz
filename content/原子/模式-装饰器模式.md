# 装饰器模式

**意图**：动态地给一个对象添加一些额外的职责。

## 解决的问题

- **接口简化**：调用方感受不到额外的功能，就像调用原函数一样
- **解耦**：日志、缓存、权限等功能与核心逻辑分离，可以独立添加或移除

## 适用场景

- 给函数添加日志
- 给类添加缓存
- 给请求添加权限检查

## 生活例子

就像手机贴膜、装手机壳，是在原有手机上额外加东西。

## 代码示例

```python
import time

def timer(func):
    """装饰器：统计函数执行时间"""
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} 执行耗时: {end-start:.4f}秒")
        return result
    return wrapper

def logger(func):
    """装饰器：记录函数调用日志"""
    def wrapper(*args, **kwargs):
        print(f"调用函数: {func.__name__}")
        result = func(*args, **kwargs)
        print(f"函数 {func.__name__} 执行完成")
        return result
    return wrapper

@timer
@logger
def fetch_data(url):
    """模拟获取数据"""
    time.sleep(0.5)  # 模拟网络请求
    return f"数据来自 {url}"

# 使用 - 原函数接口不变，功能可以叠加
result = fetch_data("https://example.com")
print(result)
# 输出:
# 调用函数: fetch_data
# fetch_data 执行耗时: 0.5000秒
# 函数 fetch_data 执行完成
# 数据来自 https://example.com
```
